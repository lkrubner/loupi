This software expects routes that look like this:

Base url is: http://das.launchopen.com/api/v0.1/

Appended to the base URL is:

GET "/:name-of-collection"

GET "/:name-of-collection/sort/:field-to-sort-by"

GET "/:name-of-collection/sort/:field-to-sort-by/:offset-by-how-many/:return-how-many"

GET "/:name-of-collection/match-field/:match-field/match-value/:match-value"

GET "/:name-of-collection/sort/:field-to-sort-by/:offset-by-how-many/:return-how-many/match-field/:match-field/match-value/:match-value"

GET "/:name-of-collection/:document-id"

POST "/:name-of-collection/:document-id"

PUT "/:name-of-collection/:document-id"

PUT "/:name-of-collection/"

DELETE "/:name-of-collection/:document-id"

Our schema is detailed at Github.

You can learn more about the API by reading the README.

For now the API does not enforce a schema, so you can create new collections on the fly. Truly, you can stuff anything you like in the database, and then get it back out later.

## Matching/Finding particular values

This route allows for matching:

GET "/:name-of-collection/match-field/:match-field/match-value/:match-value"

So we can match against a field, seeking a particular value. So for instance, to get all records where "gender" is "Female":

	    $.ajax({
            type: "GET",
            contentType: "application/json",
            url: "http://das.launchopen.com/api/v0.1/User/match-field/gender/match-value/Female",
	    data: "",
            complete: function (data) {
            var rawResponse = data.responseText;
            var cleanedData = rawResponse.replace(/\\/g, '');
            $("#api-results").text(cleanedData);
  	    alert("Done. Use FireFox for testing and look at the console.log to see the results.");  
	    },
            success: function (data) { console.log(data);  },
            error:  function (data) { console.log(data); },
            dataType: "json"
            });

This route allows even more:

GET "/:name-of-collection/sort/:field-to-sort-by/:offset-by-how-many/:return-how-many/match-field/:match-field/match-value/:match-value"

This allows us to sort, paginate, and match against a value. So here is 1,000 users whose "role" is "teacher", sorted by when they last logged-in: 


	    $.ajax({
            type: "GET",
            contentType: "application/json",
            url: "http://das.launchopen.com/api/v0.1/User/sort/logged-in-at/0/1000/match-field/role/match-value/teacher",
	    data: "",
            complete: function (data) {
            var rawResponse = data.responseText;
            var cleanedData = rawResponse.replace(/\\/g, '');
            $("#api-results").text(cleanedData);
  	    alert("Done. Use FireFox for testing and look at the console.log to see the results.");  
	    },
            success: function (data) { console.log(data);  },
            error:  function (data) { console.log(data); },
            dataType: "json"
            });

This app is not meant to be a full-featured search app, for that you should query Evan's search app. But this app does let you match any field you want.

Troubleshooting

If you get a 415 Unsupported Media Type response then make sure you set contentType: "application/json"

If you get a 404 but the URL looks correct to you, then check the verb: GET, PUT, POST, DELETE. The routes are specific to a verb and not all routes support GET. Also, avoid trailing slashes at the end of the URL.

If you get a 502 error then the loupi app has crashed and needs to be restarted.
